# Preflight

Mark every applicable item pass or fix it before delivery. Mark `NA` only with a reason.

## Brief and hierarchy

- [ ] The Design Read and three dials are stated and reflected in the result.
- [ ] The first viewport explains subject, value, and next action.
- [ ] One primary action is clear at each decision point.
- [ ] Claims have relevant proof or have been softened.

## Visual direction

- [ ] Color, shape, and theme locks are coherent across the page.
- [ ] Typography and image treatment fit the brief.
- [ ] Repeated sections use intentional variation, not default equal-card grids.
- [ ] Eyebrows are scarce and useful.
- [ ] Shipped copy and code comments contain zero em dashes.
- [ ] Images are real, licensed, supplied, generated, or clearly labeled placeholders.

## Quality and inclusion

- [ ] Mobile and wide layouts preserve hierarchy and readable line lengths.
- [ ] Contrast, keyboard access, focus visibility, semantics, and alt text are checked.
- [ ] Motion is motivated and has a reduced-motion path.
- [ ] Fonts, images, and dependencies use the project-approved pipeline.

## Redesign only

- [ ] Tokens, IA, SEO, routes, and conversion paths were audited before changes.
- [ ] Metadata, canonical behavior, structured data, and essential interactions remain intact unless explicitly changed.
